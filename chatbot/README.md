# Chatbot

A natural-language interface to the Workforce & Industry Trend Analyzer's outputs. Instead of opening a dashboard and hunting for a filter, you ask a question — "which department has the highest attrition risk?" — and get an answer pulled directly from the scored data.

This isn't a separate project bolted on for the sake of having a chatbot. It sits on top of the same CSVs the dashboards already use (`workforce_with_ml_results.csv`, `callcenter_sentiment.csv`), so anything the Random Forest, K-Means, and VADER models already computed is queryable in plain English instead of requiring someone to know which dashboard tab to click.

---

## What it's for

The dashboards are built for people who want to explore. This is built for people who just want the answer — a manager who wants to know "who's at risk on my team" without learning Tableau, or a quick sanity check while you're already in a meeting.

---

## How to run

From the repo root:

```bash
pip install -r requirements.txt
streamlit run chatbot/app.py
```

This opens a local Streamlit app in your browser where you can ask questions directly against the scored datasets.

---

## Example queries

- "Which department has the highest predicted attrition risk?"
- "Show me employees in the high-risk, low-satisfaction segment."
- "What's the average sentiment score for calls this week?"
- "How many employees are flagged as high risk overall?"

---

## How it fits into the larger project

```
Scored Output (CSVs)
   ├── Tableau Dashboards
   ├── Plotly HTML Dashboards
   └── Chatbot (chatbot/app.py)   ← this folder
```

It's a fourth way to consume the same pipeline output — no separate model, no separate data source. If the underlying CSVs get regenerated by re-running the ML pipeline, the chatbot's answers update with them.

---

## Notes

Like the rest of the project, this runs against the synthetic datasets described in the main [README](../README.md#notes-on-the-data) — the interface and retrieval logic are fully functional, the underlying records are not from a live company system.
